2.1  HostPoolResourceCompat.cs
     Purpose: Maps old method names to new generated names:
              - ImportAppAttachPackageInfos -> Import (wraps old request type)
              - ExpandMsixImages -> Expand (wraps old request type)
              - GetPrivateLinkResources -> GetByHostPool
     Can remove? NO - breaking change. These were public API methods.
     TypeSpec fix? No. The spec renamed the operations; these are SDK compat shims.
     Issues: None.

2.2  VirtualApplicationGroupResource.cs
     Purpose: Maps old GetStartMenuItems(CancellationToken) to itself.
     Can remove? NO - breaking change.
     TypeSpec fix? No.
     Issues: *** BUG - INFINITE RECURSION ***
             GetStartMenuItemsAsync(CancellationToken cancellationToken) calls
             GetStartMenuItemsAsync(cancellationToken), which resolves back to
             itself (C# overload resolution picks the 1-param overload as the
             best match because the 4-param overload's first positional param is
             int?, not CancellationToken).
             SAME BUG for the sync GetStartMenuItems(CancellationToken).
             This will cause a StackOverflowException at runtime.
             FIX: Should use named parameters to route to the 4-param overload:
               => GetStartMenuItemsAsync(pageSize: null, isDescending: null,
                     initialSkip: null, cancellationToken: cancellationToken);

2.3  VirtualApplicationGroupResourceCompat.cs
     Purpose: Maps old GetStartMenuItems(pageSize,isDescending,initialSkip,ct)

3.3  ScalingPlanData.cs
     Purpose: Two customizations:
              a) Ctor ScalingPlanData(AzureLocation) - old ctor without timeZone.
                 Defaults TimeZone to "Central Standard Time". Generated ctor
                 now requires (AzureLocation, string timeZone).
              b) Obsolete HostPoolType property (HostPoolType?) mapping to
                 ScalingHostPoolType (renamed in spec from HostPoolType to
                 ScalingHostPoolType).
     Can remove? NO - breaking changes for both.
     TypeSpec fix? No.
     Issues: The HostPoolType property getter does ToString() on ScalingHostPoolType
             and relies on implicit conversion from string to HostPoolType? - this
             works but is fragile.

     Purpose: Obsolete HostPoolType property.
     Can remove? NO - breaking change.
     TypeSpec fix? No.
     Issues: None.

3.5  Models/ScalingSchedule.cs
     Purpose: Provides 4 obsolete DateTimeOffset? properties (RampUpStartOn,
              PeakStartOn, RampDownStartOn, OffPeakStartOn) that map to/from the
              new ScalingActionTime type properties (RampUpStartTime, etc.).
     Can remove? NO - breaking change.
     TypeSpec fix? No.
     Issues: *** POTENTIAL BUG ***
             The getter creates DateTimeOffset with year/month/day = 0:
               new DateTimeOffset(0, 0, 0, Hour, Minute, 0, new TimeSpan())
             DateTimeOffset constructor expects year >= 1, month >= 1, day >= 1.
             This will throw ArgumentOutOfRangeException at runtime.
             FIX: Should use a valid date, e.g.:
               new DateTimeOffset(1, 1, 1, Hour, Minute, 0, TimeSpan.Zero)
             (Though since this is marked [Obsolete], it may be intentionally
             left semi-broken to discourage use.)

SUMMARY OF ISSUES FOUND
================================================================================

1. *** CRITICAL BUG *** VirtualApplicationGroupResource.cs (lines 38, 56):
   GetStartMenuItemsAsync(CancellationToken) and GetStartMenuItems(CancellationToken)
   recursively call themselves, causing StackOverflowException at runtime.
   FIX: Use named parameters or call GetAllAsync/GetAll directly.

2. *** POTENTIAL BUG *** ScalingSchedule.cs (lines 19, 29, 39, 49):
   DateTimeOffset constructors with year=0, month=0, day=0 will throw
   ArgumentOutOfRangeException. These are marked [Obsolete] so impact is limited
   to users ignoring the warning.

3. MINOR: ArmDesktopVirtualizationModelFactoryCompat.cs line 60 has a garbled
   XML doc comment for the iconContent parameter.
